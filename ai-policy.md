# AI Policy + Expectations

This course treats AI tools as learning aids, not substitutes for your own reasoning.
You may use tools such as Claude Code, ChatGPT, GitHub Copilot, or similar systems to help you write, run, debug, and revise code, but you remain responsible for the design, interpretation, and verification of every result you submit.

## What is allowed

You may use AI tools to:

- debug code and interpret error messages
- prototype code and refactor for clarity
- generate tests and sanity checks
- ask for conceptual explanations
- improve reproducibility and project organization
- check grammar, spelling, or formatting

## What is required

For each assignment or project, you should disclose AI use briefly and clearly:

- what you asked the tool to help with
- what you accepted, changed, or rejected

You are also responsible for verification.
If you cannot explain, defend, or reproduce a result, it should not appear in submitted work.

## What is not allowed

- submitting AI-written narrative as your own without substantial original reasoning
- turning in analyses you cannot explain
- using fabricated citations, references, or results
- using AI during any closed-tool assessment unless explicitly permitted

## Data and privacy

- do not paste sensitive or private data into external AI tools
- keep your work in a dedicated course project folder or repository
- treat all AI output as potentially wrong until you verify it

## Course note

This repository includes the public course AI policy for `PLSC 40601`.
Individual assignments may add more specific instructions or verification requirements.
